LLM Context Mutation — Terminology
Hydrate
Load structured data into the context window. Common in RAG and agent systems — implies filling from a durable source.
Inject
Put data into context. Accurate but sounds passive — doesn't convey the ongoing lifecycle.
Populate
Fill context with structured entries. Clean and readable, but still implies a one-time operation.
Maintain / Manage
Ongoing selective add / remove lifecycle — the most precise term for how context evolves across the planning session.
ITA uses Hydrate + Maintain — context is initially hydrated from Tax Plan schema, then selectively maintained as the planning session evolves.

4 LLM Engine Layer
Three engines, each consuming canonical inputs and producing form-conformant outputs
⚡ Shadow Mode vs. ATE — Test Upfront
Engine 1
LLM Tax Engine
Projection calculation and tax law application. Consumes the canonical input schema — produces Tax Calculation Results in the canonical output schema at form-line level (1040, 1120-S, 1065).
Engine 2
LLM Strategy Applicability
Determines which strategies apply to this specific filer. Chains by dependency, ranks by after-tax dollar impact, filters inapplicable options, discloses audit risk with case citations.
Engine 3
LLM Scenario Planning
Baseline vs. strategy comparison, multi-year modeling, what-if scenarios (income changes, elections, life events). Output always expressed at form-line level. Plain-language + pro-detail toggle.
⚡ Shadow Mode — Accuracy Gate (Test Upfront)
All three LLM engines run in parallel with ATE (the existing Intuit calculation engine) before any live exposure. Every result is compared on every calculation. ITA 2.0 takes primary only after sustained accuracy validation — accuracy is tested upfront, not discovered in production.
